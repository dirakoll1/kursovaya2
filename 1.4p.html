1.4. Модель зрелости Ричардсона



Теперь же рассмотрим, как определить является ли созданный вами API полностью RESTful или нет? Для этого существует модель зрелости Ричардсона, которая обычно используется для анализа того, насколько RESTful является ваш API.

Модель зрелости Ричардсона – это популярная модель, используемая для ранжирования вашего API на основе проверок, связанных с REST. Чем больше ваш API соответствует проверкам и ограничениям, тем более RESTful ваш API для разработки и развертывания. Эта специальная модель имеет четыре стадии, которые называются уровнями, и уровни варьируются от 0 до 3. [7]

Если ваш API находится на уровне 0, это означает, что ваш API совсем не RESTful, а с другой стороны, если ваш API находится на уровне 3, то можно сказать, что ваш API полностью RESTful и выполняет все ограничения RESTfulness. Давайте посмотрим, что каждый уровень содержит в себе.

1. Уровень 0


Нулевой уровень зрелости не использует ни одну из возможностей URI, HTTP-методов и HATEOAS. Службы на нулевом уровне зрелости имеют единственный URI и используют единственный метод HTTP (обычно POST).



2. Уровень 1


Первый уровень зрелости использует URI, но не использует методы HTTP и HATEOAS. Сервисы первого уровня зрелости используют множество URI, но только один HTTP-глагол - обычно HTTP POST.

Эти сервисы присваивают каждому ресурсу, доступному в системе, уникальный URI. Уникальный URI отдельно идентифицирует один уникальный ресурс - и это делает эти сервисы лучше, чем сервисы нулевого уровня.



3. Уровень 2


Второй уровень зрелости использует URI и методы HTTP, но не использует HATEOAS. Сервисы второго уровня обычно содержат множество URI, т.е. адресуемых ресурсов.

Такие сервисы поддерживают несколько глаголов HTTP на каждом открытом ресурсе - сервисы Create, Read, Update и Delete (CRUD). Здесь можно манипулировать состоянием ресурсов, обычно представляющих бизнес-сущности, по сети.



4. Уровень 3


Третий уровень зрелости использует все три принципа, т.е. URI, HTTP и HATEOAS. Третий уровень - это наиболее зрелый уровень модели Ричардсона, который поощряет легкую обнаруживаемость. На этом уровне ответы легко становятся самоописательными благодаря использованию HATEOAS. Услуги третьего уровня ведут потребителей услуг по следу ресурсов, вызывая в результате переходы состояния приложения.
